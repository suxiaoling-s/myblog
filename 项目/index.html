    项目文档¶

    这里存放各个项目的技术文档和学习总结。

    目录结构¶

    项目/
├── RadarLive-CN/          # RadarLive-CN 项目
│   ├── 项目结构总览.md
│   ├── 后端架构详解.md
│   ├── 前端架构详解.md
│   ├── API接口文档.md
│   └── 学习收获.md
└── 项目名称/              # 新项目（按此结构添加）
    ├── 项目结构总览.md
    ├── 技术文档.md
    └── 学习收获.md

    添加新项目¶

    步骤 1：创建项目目录¶

    在 docs/项目/ 下创建新项目文件夹，例如：

    mkdir -p docs/项目/项目名称

    步骤 2：创建项目文档¶

    在新项目目录下创建文档文件，建议包含：

    • 项目结构总览.md - 项目整体介绍和目录结构
    • 技术文档.md - 技术实现细节（可按需细分）
    • 学习收获.md - 项目开发过程中的收获和心得

    步骤 3：更新导航配置¶

    编辑 mkdocs.yml 文件，在 nav 部分的 项目: 下添加新项目：

    nav:
  - 首页: index.md
  - 项目:
    - RadarLive-CN:
      - 项目结构总览: 项目/RadarLive-CN/项目结构总览.md
      # ... 其他文档
    - 新项目名称:                    # 添加这里
      - 项目结构总览: 项目/新项目名称/项目结构总览.md
      - 技术文档: 项目/新项目名称/技术文档.md
      - 学习收获: 项目/新项目名称/学习收获.md

    步骤 4：更新首页¶

    编辑 docs/index.md，在项目文档部分添加新项目的链接。

    步骤 5：提交并推送¶

    git add .
git commit -m "添加新项目：项目名称"
git push
