# MkDocs + Material 博客部署指南

## 第一步：本地安装和测试

### 1. 安装依赖

```bash
cd blog
pip install -r requirements.txt
```

### 2. 本地预览

```bash
mkdocs serve
```

访问 http://127.0.0.1:8000 查看效果

### 3. 编辑内容

编辑 `docs/学习总结/RadarLive-CN项目学习收获.md` 文件，写你的学习收获。

## 第二步：创建 GitHub 仓库

### 1. 在 GitHub 上创建新仓库

- 仓库名：`blog`（或你喜欢的名字）
- 设置为 Public（GitHub Pages 免费版需要）
- 不要初始化 README

### 2. 修改配置文件

编辑 `mkdocs.yml`，修改以下内容：

```yaml
site_name: 你的博客名称
site_author: 你的名字
site_url: https://你的用户名.github.io/blog  # 修改为你的仓库地址

repo_name: 你的用户名/blog  # 修改为你的仓库
repo_url: https://github.com/你的用户名/blog  # 修改为你的仓库地址
```

### 3. 提交代码到 GitHub

```bash
cd blog

# 初始化 git（如果还没有）
git init

# 添加文件
git add .

# 提交
git commit -m "Initial commit"

# 添加远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/blog.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

## 第三步：部署到 GitHub Pages

### 方法一：使用 GitHub Actions（推荐）

1. 代码已经包含 `.github/workflows/ci.yml` 文件
2. 推送代码后，GitHub Actions 会自动构建和部署
3. 在仓库 Settings → Pages 中，选择 Source 为 `gh-pages` 分支
4. 等待几分钟，访问 `https://你的用户名.github.io/blog`

### 方法二：手动部署

```bash
# 安装 mkdocs
pip install mkdocs-material

# 部署到 GitHub Pages
mkdocs gh-deploy
```

## 第四步：后续更新

### 1. 编辑文章

编辑 `docs/学习总结/RadarLive-CN项目学习收获.md` 或其他 Markdown 文件

### 2. 本地预览

```bash
mkdocs serve
```

### 3. 提交和推送

```bash
git add .
git commit -m "更新文章内容"
git push
```

### 4. 自动部署

如果使用 GitHub Actions，推送后会自动部署。如果手动部署：

```bash
mkdocs gh-deploy
```

## 常见问题

### 1. 本地预览时样式不对

确保安装了所有依赖：
```bash
pip install -r requirements.txt
```

### 2. GitHub Pages 显示 404

- 检查仓库 Settings → Pages 中 Source 是否设置为 `gh-pages` 分支
- 等待几分钟让 GitHub 完成部署
- 检查 `mkdocs.yml` 中的 `site_url` 是否正确

### 3. 如何添加新文章

1. 在 `docs/` 目录下创建新的 Markdown 文件
2. 在 `mkdocs.yml` 的 `nav` 部分添加导航链接
3. 提交并推送
